## Interview Talk Track

Most RAG demos only show that a system can answer a question. This project shows how I would evaluate and govern an Arabic RAG system before release. It wraps the RAG backend with an adapter, runs a golden Arabic evaluation suite, validates citations and source grounding, checks safety and Arabic quality, records structured artifacts, and converts results into release decisions.

The important part is that the gates are active: one demo promotes a good candidate, one sends degraded quality to human review, and one blocks a candidate with broken citations.
